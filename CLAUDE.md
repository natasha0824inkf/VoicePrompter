# Project conventions

## Writing style
- No em dashes (—) in any markdown files. Use ` - ` (space-dash-space) instead.

## Branch naming
- Use descriptive names like `feat/offline-voice`, not auto-generated ones like `claude/lucid-curie-xv68j0`.

## Commits
- No `Co-Authored-By` or Claude session URLs in commit messages.
